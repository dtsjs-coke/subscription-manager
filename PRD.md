# PRD (Product Requirements Document)
# 구독 서비스 관리 웹페이지 v2.0

## 1. 개요

### 1.1 프로젝트 목적
여러 사용자가 각자의 구독 서비스를 독립적으로 관리하고, 만료일 전에 본인의 텔레그램으로
알림을 받을 수 있는 웹 기반 구독 관리 도구를 개발한다.

### 1.2 목표 사용자
다수의 개인 사용자 (각자 독립적인 구독 목록 보유, 셀프 회원가입 구조)

---

## 2. 기능 요구사항

### 2.1 인증 기능

#### 2.1.1 회원가입
- 웹에서 직접 회원가입 가능
- 입력 항목: 아이디, 비밀번호, 텔레그램 Chat ID (선택)
- 아이디 중복 확인 후 즉시 로그인 가능

#### 2.1.2 로그인 / 로그아웃
- 아이디 + 비밀번호 입력으로 로그인
- 로그인 성공 시 본인 구독 관리 페이지로 이동
- 세션 기반 로그인 유지 (브라우저 탭 내)

#### 2.1.3 내 정보 수정
- 비밀번호 변경 가능
- 텔레그램 수신 정보 수정 가능 (Chat ID / @username)

### 2.2 구독 관리 기능

#### 2.2.1 구독 목록 조회
로그인한 사용자 본인의 구독 항목만 표시. 표시 컬럼:

| 컬럼명 | 설명 |
|---|---|
| 서비스명 | 구독 서비스 이름 |
| 구독 시작일 | 구독 시작일 |
| 구독 만료일 | 구독 종료일 |
| 결제일 | 매월/매년 결제가 이루어지는 날짜 |
| 구독 기간 | 월간 / 연간 등 |
| 총 가격 | 구독 총 금액 |
| 월 환산 가격 | 월별 환산 비용 |
| 공유 인원 | 함께 사용하는 인원 수 |
| 관련 링크 | 서비스 URL |
| 설명 | 구독 관련 메모 |
| 비고 | 기타 사항 |

만료 임박 항목 색상 강조: D-7 이하 빨강, D-30 이하 주황

#### 2.2.2 구독 CRUD
- 웹 UI에서 추가 / 수정 / 삭제
- 변경 사항은 GitHub API로 YAML에 즉시 커밋

### 2.3 텔레그램 알림

- 만료 30일 전 / 7일 전 / 1일 전 / 당일, 각 사용자 텔레그램으로 개별 발송
- 텔레그램 정보 없는 사용자는 발송 생략
- GitHub Actions cron으로 매일 KST 09:00 실행

### 2.4 요약 정보
본인의 활성 구독 수, 월간 총액, 연간 총액 표시

---

## 3. 비기능 요구사항

| 항목 | 내용 |
|---|---|
| 보안 | 비밀번호 bcrypt 해싱 저장, Bot Token은 Streamlit Secrets 관리 |
| 데이터 영속성 | GitHub Contents API로 YAML 커밋 (Streamlit Cloud 파일시스템 우회) |
| 사용자 격리 | 본인 데이터만 조회/수정/삭제 가능 |
| 배포 | Streamlit Cloud + GitHub 자동 배포 |

---

## 4. 제약사항
- 별도 DB 미사용 (YAML 파일 기반)
- Streamlit Cloud 파일시스템 쓰기 불가 → GitHub API 우회
- GitHub API Rate Limit: 인증 시 5,000건/시간

---

## 5. 성공 지표
- 회원가입 / 로그인 / 로그아웃 정상 동작
- 사용자별 독립적인 구독 목록 관리 및 GitHub 커밋 반영
- 각 사용자 텔레그램으로 만료 알림 수신
- 앱 재배포 후에도 데이터 유지
